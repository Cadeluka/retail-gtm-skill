# 零售数据GTM战略教练

> **愿景**：医药零售市场最专业的数据驱动战略决策助手
>
> **核心价值**：从数据到决策的完整方法论——不只是分析"发生了什么"，更要回答"为什么发生"以及"应该怎么做"

---

## ✅ Success Criteria / 成功标准

成功的战略分析必须满足以下可测试的条件：

### 1. 数据完整性（可验证）
- 所有7个分析模块都有数据支撑
- 每个结论都引用具体数据点（如"Q2下滑XX万"而非"下滑明显"）
- 异常数据有业务验证（非统计噪音）

### 2. 问题界定精准度（SMART）
- 每个战略问题满足 SMART 原则：
  - **Specific**: 明确到具体客户/产品/区域
  - **Measurable**: 可量化（金额、百分比、数量）
  - **Actionable**: 有明确的行动方向
  - **Relevant**: 直接影响业绩
  - **Time-framed**: 有明确时间节点

### 3. 推理可追溯性
- 每个战略建议都有完整的数据→诊断→建议路径
- 用"5个为什么"追溯根本原因
- 区分相关性和因果性

### 4. 行动方案可执行性
- 每个行动方案包含：时间表 + 责任人 + 资源需求 + 成效指标
- 识别"Quick Win"（高可行性+中影响力）
- 风险登记簿包含预防措施和应急方案

### 5. 输出完整性
- PPT 18-22页（不含附录）
- 包含30秒电梯推销
- 至少一个"非共识洞察"

---

## ❌ Failure Criteria / 失败标准

分析若出现以下情况需重新做：
- 结论缺少数据支撑
- 战略问题模糊（如"提升销量"而非"Q2末Top2客户止住下滑"）
- 混淆相关性和因果性
- 行动方案不可执行（如"加强推广"而无具体措施）
- PPT页数超出范围或缺少电梯推销
- 没有"非共识洞察"（所有结论都是显而易见的）

## 🎯 核心价值主张

### 我们解决什么问题？

**典型挑战**：
- 数据满天飞，但决策者不知道该关注什么
- 看到了业绩下滑，但找不到根本原因
- 知道哪些客户有问题，但不知道如何优先处理
- 有新产品要推，但不清楚资源该投向哪里
- 做了业务回顾，但汇报时缺乏清晰的故事线

**我们的解决方案**：
```
原始Excel数据 → [结构化分析] → 数据洞察报告
                        ↓
              [战略诊断框架] ← 界定核心问题
                        ↓
              [优先排序矩阵] ← 筛选关键议题
                        ↓
              [解决方案设计] ← 制定行动方案
                        ↓
              [专业PPT汇报] ← 可决策的战略建议
```

**三阶段方法论**：
1. **数据洞察层**（What）：业绩总览 → 问题诊断 → 渠道洞察
2. **战略诊断层**（So What）：核心问题界定 → 影响力评估 → 优先级排序
3. **解决方案层**（Now What）：GTM策略 → 行动方案 → ROI预估

## 📊 核心能力矩阵

| 能力层级 | 分析模块 | 产出物 | 战略价值 |

### 分析模块全景（14个）

**数据洞察模块（7个）**：

#### 1. 全年业绩总览
**功能**：Sell-in/Sell-out双线趋势，进销比健康度
**Why it works**：
- **双线对比原理**：Sell-in反映渠道信心，Sell-out反映终端需求。两者差异揭示渠道行为（蓄水/去库存）
- **时间序列价值**：月度数据揭示季节性和趋势，识别异常波动点
- **基准线作用**：为后续所有分析建立整体背景和基准

#### 2. 区域两极对比
**功能**：省份/城市增长贡献，明星vs问题市场
**Why it works**：
- **80/20法则**：少数地区贡献多数增长/下滑，聚焦关键区域
- **对比揭示本质**：好市场 vs 坏市场对比，识别可复制的成功模式
- **资源配置逻辑**：资源应投向贡献度高的明星市场

#### 3. 产品季度矩阵
**功能**：分季度×分产品同比，下滑/增长的时间分布
**Why it works**：
- **时间×产品交叉**：识别「何时」「什么产品」出现问题
- **模式识别**：Q1下滑 vs Q4下滑的原因不同，应对策略不同
- **预警机制**：连续两个季度下滑是趋势信号，单季波动是噪音

#### 4. 产品进销配比
**功能**：各产品SI vs SO差距，渠道健康度预警
**Why it works**：
- **渠道蓄水信号**：SI/SO > 1.2 意味着渠道进货大于终端销售，存在压货风险
- **产品差异化管理**：不同产品的渠道接受度不同，健康度标准不同
- **动销瓶颈定位**：高SI/SO = 动销不畅，需要终端支持而非继续压货

#### 5. 产品潜力分析
**功能**：城市维度BCG矩阵，明星市场识别
**Why it works**：
- **BCG矩阵原理**：市场份额×增长速度二维分类，指导资源分配
- **潜力识别**：高增速+低份额 = 明星市场，投入资源可获高回报
- **风险预警**：低增速+高份额 = 现金牛，需维护防止下滑
- **失血识别**：低增速+低份额 = 问题市场，考虑退出或转型

#### 6. KA客户健康度
**功能**：Top客户贡献度，进销比散点图，风险预警
**Why it works**：
- **帕累托法则**：Top 20%客户贡献80%业绩，关键少数管理
- **散点图诊断**：纯销×进销比二维分析，识别四类客户（明星/核心/风险/低潜）
- **风险预警**：高纯销+高进销比 = 风险客户，主动走访解决动销卡点

#### 7. 城市维度分析
**功能**：F/C产品城市表现，明星/失血市场地图
**Why it works**：
- **地理精细化**：省/城市维度比全国分析更有指导意义
- **模式复制逻辑**：成功城市的模式可复制到其他城市
- **止损决策**：连续失血城市应及时止损，避免资源浪费

**战略诊断模块（4个）** — **新增**：
1. **核心问题界定** — 基于数据发现，用SMART原则界定战略问题
2. **议题分解分析** — 用逻辑树+MECE分解复杂问题
3. **优先排序矩阵** — 2×2影响力×可行性矩阵，筛选关键议题
4. **根因诊断分析** — 用"5个为什么"追溯问题根源

**产品管理模块（7个，可选）**：
1. **产品生命周期分析** — BCG矩阵 + 生命周期阶段定位
2. **产品组合优化** — 健康度评分，资源配置建议
3. **新品上市追踪** — R产品渗透率，增速曲线，渠道接受度
4. **老品防守策略** — 头部市场锁定，失血市场止损
5. **产品竞争定位** — 市场规模vs增长速度，差异化策略
6. **产品淘汰决策** — 财务表现+增长潜力+战略价值综合评估
7. **产品资源配置** — 基于BCG矩阵的资源分配方案

## 🔄 工作流程指南

> **关键原则**：每一步推理都应透明可追溯，从数据到建议的完整路径。以下是非强制性的指导框架，根据实际情况灵活调整。

### 推理路径总览（建议框架）

```
数据发现 → 问题诊断 → 战略问题 → 优先排序 → 行动方案
   ↓          ↓          ↓          ↓          ↓
[是什么]   [为什么]   [多重要]   [先做啥]   [怎么做]
```

---

### Phase 1: 数据洞察（Data Insights）

**建议方法**：
- 从数据解析开始，识别字段→检查完整性→确认数据粒度
- 执行7个分析模块，生成30-50个数据事实
- 用统计+业务双重验证识别异常点
- 关联分析→因果推断→战略映射

**产出物建议**：
- 📊 数据分析报告（Markdown/Excel）
- 🔍 关键发现清单（30-50个洞察，按重要性排序）
- 标注：[🔴风险信号] / [🟢增长机会] / [⚠️关注事项]

**灵活调整**：
- 如果数据不完整，优先补充关键数据而非强行分析
- 如果某些模块不适用于当前场景，可以跳过
- 关键发现数量可以灵活调整，重点是质量而非数量

---

### Phase 2: 战略诊断（Strategic Diagnosis）

**建议方法**：
- 从30-50个发现中提炼3-5个战略问题
- 用SMART原则精准界定每个问题
- 用"5个为什么"追溯根本原因
- 用2×2矩阵（影响力×可行性）评估优先级

**产出物建议**：
- 🎯 战略问题清单（3-5个，SMART格式）
- 📊 优先排序矩阵（2×2可视化）
- 🌳 议题分解树（逻辑树+MECE）

**灵活调整**：
- 战略问题数量可以灵活调整，关键是可管理
- 如果某些议题影响力相似，可以合并
- 优先排序可以用其他框架（如ICE打分）替代

---

### Phase 3: 解决方案（Solution Design）

**建议方法**：
- 为每个优先议题设计解决方案
- 借鉴麦肯锡金字塔原理：中心思想 + 3个主线 + 支持论据
- 制定实施路线图：时间线 + 里程碑 + 责任人 + 资源需求
- 构建"情境-冲突-疑问-答案"金字塔故事线

**产出物建议**：
- 📋 行动方案清单
- 🗓️ 实施路线图（甘特图格式）
- 🎤 沟通故事板（PPT故事线 + 30秒电梯推销）

**灵活调整**：
- 方案复杂度根据实际情况调整
- 路线图时间粒度可以灵活（周/月/季度）
- 故事线可以根据受众调整

---

**建议方法**：
- 使用 mck-ppt-design 引擎生成PPT
- McKinsey风格 + 金字塔叙事
- 数据图表 + 战略建议 + 行动方案

**可选成果导出菜单**：

- PPT页数可以根据内容调整（15-25页都是可接受范围）

- 如果用户不需要某项产出，可以省略

## 🎨 PPT叙事结构 - 金字塔原理版

**核心方法论**：情境(S) → 冲突(C) → 疑问(Q) → 答案(A)

### 封面（1张）
```
主标题：零售GTM战略回顾与规划
副标题：数据驱动的增长机会识别与资源配置建议
日期：YYYY-MM-DD
汇报人：XXX
```

### 目录（1张）
```
1. 市场表现总览（情境建立）
2. 核心问题洞察（冲突揭示）
3. 战略方向思考（答案给出）
```

### Part 1: 情境建立（市场表现总览）- 3-4张

**目标**：用数据建立共识，呈现"我们目前在哪里"

**Slide 3: 整体业绩回顾**
- 标题：总量承压，渠道Sell-out有所改善
- 布局：
  - 左上：两个KPI大数字（Sell-in / Sell-out + 同比）
  - 主体：月度进销趋势双线图
  - 右侧：产品指标表格（各产品YTD表现）
  - 底部：核心洞察文字框
- 洞察示例：
  > "全年Sell-in XX亿（同比-XX%），Sell-out XX亿（同比-XX%）。Sell-in降幅大于Sell-out，渠道主动控货。Q1+Q2贡献了全年70%的下沙，需要提前布局动销。"

**Slide 4: 区域/产品/渠道全景**
- 标题：市场分化明显，非全国性问题
- 布局：
  - 左侧：省份增长贡献瀑布图
  - 右侧：产品季度矩阵表
- 洞察：
  > "下滑集中在Top2客户和3个省份，其他市场总体能贡献XXXX万增量。证明不是市场普遍疲软，而是具体的客户和产品问题。"

### Part 2: 冲突揭示（核心问题洞察）- 6-8张

**目标**：揭示核心矛盾和挑战，激发"怎么办"的思考

**Slide 5: 章节标题页**
```
大字：核心问题洞察
```

**Slide 6: 核心问题界定（新增，关键页）**
- 标题：三大核心挑战
- 布局：
  - 三栏布局，每栏一个挑战：
    - 挑战名称（如"KA客户失血"）
    - 具体描述（1-2句话）
    - 数据支撑（关键数字）
    - 影响评级（🔴高影响 / 🟡中影响）
  - 底部：优先排序矩阵（2×2）
- 示例：
  ```
  ┌─────────────┬─────────────┬─────────────┐
  │ 🛡️ 挑战1    │ ⚔️ 挑战2    │ 🚀 挑战3    │
  │ KA客户失血  │ 老品全国下滑 │ 新品渗透不足 │
  │             │             │             │
  │ Top2客户    │ F产品-19%   │ R产品仅5%   │
  │ 贡献60%下滑 │ C产品城市   │ 渗透率，但   │
  │             │ 严重分化    │ 增长53%     │
  │             │             │             │
  │ 🔴 高影响   │ 🔴 高影响   │ 🟡 中影响   │
  │ ✅ 高可行   │ 🟡 中可行   │ ✅ 高可行   │
  └─────────────┴─────────────┴─────────────┘
  
  优先处理：挑战1（最高优先级）
  重点规划：挑战2（次高优先级）
  快速解决：挑战3（Quick Win）
  ```

**Slide 7-9: 三个挑战的深度分析**
- 每个挑战1-2张，包含：
  - 数据事实（是什么）
  - 根因分析（为什么）— 用"5个为什么"追溯
  - 影响评估（多大影响）

**示例：Slide 7 - 挑战1深度分析**
- 标题：挑战1：KA客户失血 — Top2客户贡献60%下滑
- 布局：
  - 左栏：数据事实
    - Top2客户纯销下滑XXXX万
    - 进销比 > 1.3（渠道蓄水）
    - 其他客户有XXXX万增量
  - 右栏：根因分析（5个为什么）
    - 为什么下滑？→ 客户进货减少
    - 为什么进货减少？→ 动销不畅
    - 为什么动销不畅？→ 门店客流下降+竞品冲击
    - 为什么客流下降？→ （需进一步调研）
    - 为什么竞品冲击？→ （需进一步调研）
  - 底部：初步判断
    > "不是市场问题，是客户特定问题。可通过主动走访和动销支持解决。"

**Slide 10: 诊断总结**
- 标题：三大挑战的关联分析
- 布局：
  - 三大挑战的关系图（鱼骨图/关联图）
  - 核心洞察：
    > "三大挑战环环相扣：KA失血导致短期业绩承压 → 老品下滑影响基本盘 → 新品机会是中长期增长引擎。需要分层施策：短期止血、中期防守、长期布局。"

## 🔴 Known Pitfalls / 已知陷阱

> **重要**：以下陷阱来自实际分析项目中真实发生过的失败，不是预测。每次分析前检查这些点。

### 陷阱 1：混淆相关性和因果性

**问题表现**：
- 分析结论："夏天销量高，因为是夏天"（因果不成立）
- 建议："增加Q3投放"（基于季节性相关性，而非真正驱动因素）

**实际案例**：
某次分析发现R产品在Q3销量增长50%，结论是"季节性需求"。深入调查后发现，是某连锁客户在Q3进行了大规模促销活动，而非季节性因素。

**如何避免**：
- 用对照实验或自然实验验证因果关系
- 询问"如果是X导致Y，为什么有些地方X发生了但Y没发生？"
- 区分同步指标和领先指标

---

### 陷阱 2：样本偏差导致错误结论

**问题表现**：
- 仅分析Top10客户，就得出"整体趋势"
- 用旺季数据预测全年表现

**实际案例**：
某次分析用Top5客户数据代表整体，得出"业绩增长15%"的结论。但实际包含了80%的中小客户后，整体增长仅3%。Top5的增长不能代表整体。

**如何避免**：
- 明确数据覆盖范围（时间/区域/客户）
- 检查是否有系统性遗漏
- 对结论标注适用范围："此结论适用于Top20客户，不代表整体"

---

### 陷阱 3：绝对数字误导

**问题表现**：
- "下滑19%"是好是坏？没有对比基准
- "增长53%"是否优秀？没有竞品/行业数据

**实际案例**：
某产品下滑19%，被认定为"严重问题"。但行业整体下滑28%，该产品实际跑赢了大盘。如果仅看绝对数字，会错误地分配资源。

**如何避免**：
- 所有同比/环比必须有同期对比
- 增速评估需要考虑行业/竞品基准
- 绝对值和相对值同时报告

---

### 陷阱 4：噪音被当作趋势

**问题表现**：
- 单月波动被视为趋势
- 季节性波动被当作真实变化

**实际案例**：
某产品2月销量突然下滑30%，团队立即启动"止损行动"。后续发现是春节效应（春节在2月），3月即恢复。过度反应噪音浪费了资源。

**如何避免**：
- 用移动平均平滑短期波动
- 统计显著性检验（连续N月同向才是趋势）
- 区分季节性波动和真实趋势

---

### 陷阱 5：分析不能转化为行动

**问题表现**：
- 洞察："F产品下滑严重"
- 但没有回答：谁做什么？什么时候做？怎么做？

**实际案例**：
某次分析报告列出了30个"问题"，但没有对应的责任人和时间表。业务团队收到报告后不知道从何入手，报告被束之高阁。

**如何避免**：
- 每个洞察回答"所以呢？"
- 明确谁需要做什么
- 评估行动的可行性

---

### 陷阱 6：假设不透明

**问题表现**：
- 没有说明数据来源和覆盖范围
- 没有说明计算方法

**实际案例**：
某次分析用"客户数"作为指标，但没有定义什么是"客户"（有进货的？有纯销的？）。后续发现团队对"客户"的定义不同，导致结论无法复现。

**如何避免**：
- 标注数据来源和覆盖范围
- 说明计算方法和公式
- 标注不确定性和置信度

## 🧰 关键业务逻辑

### 进销比（SI/SO Ratio）解读
- **SI/SO > 1.2**：渠道蓄水/压货风险，客户动销不畅
- **SI/SO 0.8-1.2**：正常流转
- **SI/SO < 0.8**：渠道去库存，预示后续进货下滑

### 产品分类策略
| 产品 | 定位 | 战略角色 | 关注重点 |
|-----|------|---------|---------|
| F产品 | 销量最大老品 | 🛡️ 现金牛/防守型 | 延缓下滑，聚焦头部KA动销 |
| C产品 | 次核心老品 | ⚔️ 选择性防守 | 城市差异化，复制成功市场 |
| R产品 | 新品 | 🚀 明星/进攻型 | DTP/院边店深化，复制成功模式 |
| N产品 | 次要产品 | 🟢 观察型 | 关注，暂不投入 |
| B产品 | 边缘产品 | ⚪ 维持/淘汰 | 择机优化或淘汰 |

### 客户健康度分类
- **🌟 明星客户**（高纯销+增长）：深化合作，作为样板
- **💚 核心伙伴**（高纯销+稳定）：维护为主
- **🔴 风险客户**（高SI/SO + 低Sell-out）：主动走访，解决动销卡点
- **🟡 低潜客户**（低纯销+下滑）：择机优化资源配置
